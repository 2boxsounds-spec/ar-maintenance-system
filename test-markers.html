🧪 Testing Steps

  1. Print or Display: Print one of the markers above, or display it on another device (phone, tablet, second monitor)
  2. Start the Server: Run npm start in your project folder
  3. Open AR App: Go to http://localhost:3000/ar
  4. Login: Use j.smith / Tech123!
  5. Allow Camera: Click "Allow" when browser asks for camera permission
  6. Point Camera: Point your webcam at the printed/displayed marker
  7. Wait for Detection: The system should detect the marker within 2-5 seconds
  8. Check Overlay: You should see "Fault Detected" text appear over the marker
  9. Interact: A fault info panel should appear at the bottom with details

💡 Tips for Best Detection

  • Use good lighting (not too bright, not too dark)
  • Keep the marker flat and still
  • Hold the camera steady, about 30-50cm from the marker
  • Avoid reflections on the marker surface
  • Try different angles if not detected immediately
  • Laptop webcams work better than phone cameras for testing

⚠️ Troubleshooting

Camera not opening? Check browser permissions and allow camera access.

Marker not detected? Try printing larger, improving lighting, or moving camera closer/further.

Black screen? Check if camera is being used by another app (Zoom, Teams, etc.).

AR not loading? Open browser console (F12) and check for errors.
